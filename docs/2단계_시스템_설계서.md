# 02. 시스템 설계서 (Architecture & Design)

본 문서는 IvansTerm의 기술 스택, 데이터 구조 및 파일 구성을 정의합니다.

---

## 1. 서비스 아키텍처
- **Frontend**: React (v18+) + TypeScript + Tailwind CSS + shadcn/ui. SPA 구조로 동적 패널 레이아웃 및 실시간 터미널 인터페이스 제공.
- **Backend**: Python 3.10+ / FastAPI. 비동기(Async) 처리로 WebSocket(터미널) 및 REST API(파일/설정) 동시 서빙.
- **Database**: SQLite3. 경량 로컬 DB로 접속 정보 및 작업 환경 저장.
- **Infrastructure**: Docker. 단일 이미지에 Frontend 빌드 + Backend를 묶어 간편 배포.

---

## 2. 데이터베이스 설계 (ERD)

### 2.1 connections (서버 접속 정보)
- `id` (PK), `name`, `host`, `port`, `username`, `auth_method`, `private_key_path`

### 2.2 workspaces (작업 환경 설정)
- `id` (PK), `name`, `layout_state` (JSON)
- `layout_state` 예시: `{ "panels": [{ "id": "p1", "type": "editor", "conn_id": 1, "path": "/home" }, ...], "split_ratio": 0.5 }`

### 2.3 transfer_history (파일 전송 이력)
- `id` (PK), `file_name`, `direction` (UP/DOWN), `status`, `created_at`

---

## 3. UI/UX 레이아웃 설계
- **기본 구조**: 좌/우 1:1 분할. 각 영역에 에디터 또는 터미널 패널을 동적으로 추가/제거 (한쪽 최대 3개, 전체 최대 6개).
- **에디터 패널**: 상부에 파일 트리 브라우저(접이식), 하부에 Monaco Editor. 파일 선택 시 SFTP로 읽어와 표시.
- **터미널 패널**: xterm.js 기반 풀 터미널 에뮬레이션. WebSocket을 통한 실시간 입출력.
- **상단 바**: 세션 탭 전환, 패널 추가(+) 버튼, 접속 관리 메뉴.
- **하단 상태바**: 현재 접속 정보 표시, 파일 전송 진행률 표시.

## 4. 통신 구조

```
┌─────────────┐         ┌─────────────────────────────────┐
│   Browser   │         │         FastAPI Backend          │
│  (React)    │         │                                  │
│             │◄──REST──►│  /api/connections  (접속 관리)   │
│  Monaco     │         │  /api/files        (파일 CRUD)   │
│  Editor     │◄──REST──►│  /api/transfer     (업/다운로드) │
│             │         │                                  │
│  xterm.js   │◄──WS────►│  /ws/terminal      (SSH 터미널) │
└─────────────┘         └─────────────────────────────────┘
                                     │
                                     │ SSH / SFTP
                                     ▼
                              ┌──────────────┐
                              │ Remote Server │
                              └──────────────┘
```

## 5. 파일 구조 (Project Structure)
```text
ivansterm/
├── backend/
│   ├── api/              # API 엔드포인트 (connections, files, transfer)
│   ├── core/             # 설정, 보안(암호화), SSH/SFTP 매니저
│   ├── db/               # SQLAlchemy 모델 정의, CRUD
│   ├── services/         # SSH/SFTP 세션 관리 서비스 로직
│   └── main.py           # FastAPI App 진입점
├── frontend/
│   ├── src/
│   │   ├── components/
│   │   │   ├── layout/   # 동적 패널 레이아웃, 상단/하단 바
│   │   │   ├── editor/   # Monaco Editor 래퍼, 파일 트리
│   │   │   ├── terminal/ # xterm.js 터미널 래퍼
│   │   │   └── common/   # 공통 UI (모달, 버튼 등)
│   │   ├── hooks/        # 커스텀 훅 (useSSH, useSFTP, usePanel 등)
│   │   ├── context/      # 전역 상태 관리 (ConnectionContext, PanelContext)
│   │   ├── App.tsx       # 메인 라우팅 및 레이아웃 컨테이너
│   │   ├── main.tsx      # React 진입점
│   │   └── index.css     # 전역 스타일
├── docs/                 # 프로젝트 문서
├── Dockerfile            # 멀티스테이지 빌드 설정
└── docker-compose.yml    # 포트 매핑 및 실행 설정
```
