# 📄 요구사항 정의서 (Software Requirements Specification)

## 1. 프로젝트 개요
*   **프로젝트명**: IvansTerm
*   **목표**: 단일 웹 화면 내에서 리모트 서버의 파일 편집(SFTP)과 명령어 실행(SSH)을 동시에 처리할 수 있는 통합 개발 환경 구축.
*   **핵심 가치**: 맥락 전환(Context Switching) 최소화, 다중 세션 관리, 리모트 작업의 가시성 확보.

## 2. 기술 스택 (Tech Stack)
*   **Frontend**: React (v18+), TypeScript, Tailwind CSS, shadcn/ui
*   **UI Components**: react-resizable-panels (레이아웃), xterm.js (터미널), monaco-editor (에디터)
*   **Backend**: Python 3.10+, FastAPI (Asynchronous)
*   **SSH/SFTP**: paramiko 또는 asyncssh
*   **Database**: SQLite3 (경량 로컬 DB)
*   **ORM**: SQLAlchemy 또는 Tortoise-ORM
*   **Communication**: WebSockets (터미널 실시간 통신), REST API (파일 및 설정 제어)

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 화면 레이아웃 및 UI
*   **기본 레이아웃**: 화면을 좌/우 1:1 분할로 시작. 각 영역에 패널을 **동적으로 추가/제거** 가능.
*   **동적 패널 분할**: 한쪽 영역(좌 또는 우)에 최대 **3개까지** 세로 분할하여 패널 배치 가능. 전체 화면 최대 **6개** 패널. (예: 좌1·좌2·좌3 : 우1, 좌1 : 우1·우2 등 자유 조합)
*   **패널 위치 이동**: 패널을 드래그하여 좌/우 영역 간 또는 영역 내에서 순서를 자유롭게 재배치 가능.
*   **유연한 리사이징**: 각 패널 사이의 경계선을 드래그하여 크기 조절 가능. 좌/우 영역 간 비율도 자유롭게 조절.
*   **패널 타입 선택**: 새 패널 추가 시 **에디터(Editor)** 또는 **터미널(Terminal)** 중 타입을 선택하여 배치.
*   **멀티 세션 탭**: 상단 탭을 통해 여러 리모트 접속 세션을 전환하며 작업 가능.

### 3.2 리모트 파일 에디터 (에디터 패널)
*   **파일 브라우저**: 리모트 디렉토리 구조를 트리 형태로 표시.
*   **실시간 편집**: 파일을 열어 수정하고 `Ctrl+S` 입력 시 SFTP를 통해 리모트 서버에 즉시 저장.
*   **동시 편집**: 여러 에디터 패널에 각각 다른 파일을 열어 대조하며 작업 가능.

### 3.3 SSH 터미널 (터미널 패널)
*   **풀 터미널 에뮬레이션**: xterm.js를 이용한 표준 쉘 환경 제공.
*   **디렉토리 동기화**: 에디터에서 선택한 디렉토리 경로로 터미널 자동 이동(선택 사항).
*   **브로드캐스트 명령**: (옵션) 여러 터미널 패널에 동일한 명령을 동시에 전송하여 일괄 처리 결과 확인.

### 3.4 파일 전송 (Up/Download)
*   **Drag & Drop 업로드**: 로컬 파일을 웹 화면(파일 트리 또는 에디터)으로 드래그하여 리모트 서버로 전송.
*   **우클릭 다운로드**: 파일 트리에서 특정 파일을 선택하여 로컬 PC로 다운로드.
*   **진행률 표시**: 대용량 파일 전송 시 상태바(Status Bar)에 진행률 표시.

### 3.5 접속 및 세션 관리
*   **접속 정보 저장**: Host, Port, Username, SSH Key/Password 정보를 SQLite에 암호화하여 저장.
*   **최근 작업 환경 복구**: 앱 재시작 시 마지막으로 열었던 파일 및 디렉토리 위치 복구.


## 4. 데이터 모델 (Database Schema)

### 4.1 connections (서버 접속 정보)
*   `id` (PK), `name`, `host`, `port`, `username`, `auth_method`, `private_key_path`

### 4.2 workspaces (작업 환경 설정)
*   `id` (PK), `name`, `layout_state` (JSON)
*   `layout_state` 필드 안에 각 패널의 `conn_id`, `path`, `type` 등을 포함하여 다중 접속 패널 구성 지원.

### 4.3 transfer_history (파일 전송 이력)
*   `id` (PK), `file_name`, `direction` (UP/DOWN), `status`, `created_at`

## 5. 단계별 개발 로드맵
*   **Phase 1**: FastAPI + xterm.js를 이용한 기본 SSH 터미널 웹 구현.
*   **Phase 2**: Monaco Editor + SFTP를 이용한 리모트 파일 읽기/쓰기 구현.
*   **Phase 3**: react-resizable-panels 적용 및 동적 패널 레이아웃(추가/제거/이동/리사이징) 완성.
*   **Phase 4**: 파일 업로드/다운로드 기능 및 SQLite 접속 정보 관리 기능 추가.
*   **Phase 5**: UI 고도화 (테마 선택, 단축키 설정 등).
