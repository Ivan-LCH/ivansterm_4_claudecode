# 06. 운영 매뉴얼

본 문서는 IvansTerm의 배포 및 운영 절차를 설명합니다.

---

## 1. 배포 패키지 준비 (개발 서버)

### 1.1 Docker 이미지 빌드
```bash
# 프로젝트 루트에서 실행
docker compose build
```

### 1.2 배포 패키징 (`.tar.gz`)

> ⚠️ **중요**: 아래 명령어는 반드시 `ivansterm/`의 **상위 디렉토리**에서 실행하세요.

```bash
cd /DATA/my_prog

# 1. Docker 이미지를 파일로 저장
# ⚠️ 이미지 이름은 docker-compose.yml의 image: 설정값으로 대체하세요 (구현 시 확정)
docker save ivansterm_app | gzip > ivansterm_image.tar.gz

# 2. 전체 요소를 하나의 파일로 압축
tar -czvf ivansterm_deploy.tar.gz \
    --exclude='ivansterm/frontend/node_modules' \
    --exclude='ivansterm/frontend/dist' \
    --exclude='ivansterm/static' \
    --exclude='ivansterm/__pycache__' \
    --exclude='ivansterm/.git' \
    --exclude='ivansterm/.env' \
    --exclude='ivansterm/*.log' \
    --exclude='ivansterm/db_data' \
    ivansterm/ \
    ivansterm_image.tar.gz
```
> **결과물**: `ivansterm_deploy.tar.gz` 📦

---

## 2. 설치 및 구동 (운영 서버)

### 2.1 압축 해제 및 이미지 로드
```bash
# 1. (업데이트인 경우) 기존 컨테이너 중지
cd ai_editor
docker compose down -v
cd ..

# 2. 압축 해제
tar -xzvf ivansterm_deploy.tar.gz

# 3. Docker 이미지 로드
docker load < ivansterm_image.tar.gz
```

### 2.2 환경 설정 (`.env`)
```bash
# 최초 설치 시에만 수행
cd ai_editor
cp .env.example .env
vi .env
# ENCRYPTION_KEY, 포트 설정 등 수정 후 저장
```

### 2.3 시스템 구동
```bash
docker compose up -d
```
> **확인**: `docker ps` 명령어로 컨테이너가 `Up` 상태인지 확인합니다.

## 4. 소스코드만 업데이트 (이미지 변경 없이)

> ⚠️ `Dockerfile`이나 `requirements.txt`가 변경된 경우에는 1~2단계(이미지 재빌드)를 사용하세요.

### 4.1 소스 압축 (개발 서버)
```bash
cd /DATA/my_prog

tar czf ivansterm_src.tar.gz \
    --exclude='ivansterm/frontend/node_modules' \
    --exclude='ivansterm/frontend/dist' \
    --exclude='ivansterm/static' \
    --exclude='ivansterm/__pycache__' \
    --exclude='ivansterm/.git' \
    --exclude='ivansterm/*.log' \
    --exclude='ivansterm/.env' \
    --exclude='ivansterm/db_data' \
    ivansterm/
```

### 4.2 운영 서버 적용
```bash
# 1. 압축 해제 (기존 소스 덮어쓰기)
cd /DATA/my_prog
tar -xzf ivansterm_src.tar.gz

# 2. 서비스 재시작
cd ai_editor
docker compose restart
```

## 5. 데이터베이스 초기화

> ⚠️ 아래 명령 실행 시 저장된 접속 정보, 작업 환경, 전송 이력이 모두 삭제됩니다.

```bash
# 1. 컨테이너 중지
docker compose down

# 2. DB 파일 삭제
rm -f ./db_data/*.db

# 3. 컨테이너 재시작 (초기 DB 자동 생성)
docker compose up -d
```
