  - [C] 6-1. 서버 연결 시 기존 tmux 세션 선택 UI 추가
        - 서버 연결 시도 시 해당 서버의 살아있는 tmux 세션 목록을 먼저 조회 (`tmux ls`)
        - 세션이 있으면 선택 다이얼로그 표시:
          - 기존 세션 목록 (세션명, 생성시간, 창 개수 등 표시)
          - "기존 세션 이어받기" 선택 → `tmux attach -t <session>`
          - "새 세션 시작" 선택 → 기존처럼 신규 tmux 세션 생성
        - 세션이 없으면 다이얼로그 없이 바로 새 세션으로 연결
        - 사이드바 Sessions 탭에서도 현재 세션의 tmux 세션 목록 확인 가능하도록 연동
  - [C] 6-1-1. 세션 목록 필터링: 현재 연결명 prefix 매칭 세션만 표시
        - `{연결명}_`으로 시작하는 세션만 표시 (같은 서버의 다른 연결명 세션 제외)

  - [C] 6-2. Close Session 시 해당 tmux 세션 자동 kill
        - IvansTerm에서 세션을 닫을 때 서버의 tmux 세션도 함께 종료
        - 브라우저 탭 닫기/새로고침은 감지 불가이므로, 사이드바 × 버튼으로 명시적 Close 시에만 적용
  - [C] 6-2-1. 추가 터미널(2/3번) tmux 세션도 함께 kill
  - [C] 6-2-2. 세션 닫기 확인 다이얼로그 추가
        - [나가기]: WebSocket만 끊음 (tmux 세션 유지, 다른 클라이언트 영향 없음)
        - [세션 종료]: tmux 세션 kill 후 닫기
        - [취소]: 닫기 취소
        - Terminal 1: `{서버명}_{MMDD-HHMM}`, Terminal 2: `-2` suffix, Terminal 3: `-3` suffix
        - Session 닫을 때 연결된 모든 tmux 세션을 배열로 관리하여 일괄 kill

  - [C] 6-3. tmux 세션 선택 모달에 '삭제' 버튼 추가

        - 클릭 시 해당 tmux 세션 kill 후 목록에서 제거

  - [C] 6-4. tmux 세션명을 사람이 읽을 수 있는 형식으로 변경
        - 현재: `ivan_1_term_1_357_fo6q` (내부 ID 조합, 구분 불가)
        - 변경: `{서버명}_{접속날짜시간}` 형식
          - 예: `archi-IVANSTERM_0330-1423`

          - 날짜시간: MMDD-HHMM (연결 생성 시각 기준)
